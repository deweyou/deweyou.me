Agent 不应把“工具不见了”当成普通生成错误，而要能识别、暂停、重载工具清单并恢复执行。

最近 OpenAI Agents Python 的更新里出现了“missing function tools 的可选恢复”和更多 tracing 导出，这类小改动说明真实 Agent 运行时最常见的问题不是模型不会想，而是执行环境、工具 schema、沙箱和上下文状态会漂移。

模型发出 tool call 时，运行时必须能把名称、参数 schema、权限和实际 handler 对上。若会话恢复、MCP 连接重启、版本升级或上下文压缩后工具表变化，模型可能调用一个当前不存在的工具。可靠系统不该直接让模型编造替代动作，也不该盲目重试；

应把它归类为“能力目录失配”，重新 list tools、刷新 schema，必要时让任务回到上一个安全检查点。

如果把 **工具缺失要可恢复** 当成检查题，你的系统里哪一段还只是靠默认假设在运转，而没有被设计成可观测、可验证、可回退的工程机制？
