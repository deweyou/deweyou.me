## 背景：子 Agent 正在从“工具调用”变成“后台执行体”

很多 Agent 平台一开始会把子 Agent 当成一个普通 tool：主 Agent 发起调用，子 Agent 在内部跑，最后返回一段结果。这个模型适合短任务，但一旦任务进入代码修改、网页调研、批量文件处理、长时间爬取、数据清洗或多步骤交付，问题就会暴露出来：用户只看到“正在执行”，主 Agent 也很难判断到底是卡住、失败、等待外部资源，还是已经完成了中间阶段。

Cloudflare Agents 0.17.0 在 agent-tool sub-agent 上加入 progress signalling 和 durable milestones，是一个很值得注意的信号。它不是简单地把子 Agent 的 stdout 转发出来，而是把“进度”本身建模成运行时事件：子 Agent 可以报告阶段、milestone 和中间状态；这些进度能被持久化，也能服务于 detached/background agent tools。

## 核心机制：progress 不是日志，而是协议事件

日志是给人排障看的，progress 是给系统调度和恢复看的。两者最大的区别在于：日志通常是 append-only 文本，语义弱；progress 应该有结构化字段、稳定任务 ID、阶段边界、时间戳、可覆盖策略和恢复语义。

一个成熟的子 Agent 运行时至少要区分几类事件：

- `started`：任务主体、输入、权限边界已经固化；
- `progress`：当前阶段、完成比例或正在等待的资源；
- `milestone`：已经产生了可引用的中间 artifact，例如草稿、补丁、报告、截图、索引；
- `blocked`：需要外部审批、缺少凭据、等待服务、或者遇到不可自动恢复条件；
- `completed` / `failed` / `cancelled`：终态必须可审计、可重试或可解释。

如果这些事件只存在于模型上下文里，压缩、重试、切换模型、gateway 重启之后都会丢。把 progress 做成 durable milestone，本质上是在说：长任务子 Agent 不再只是一次函数调用，而是有生命周期的执行实体。

## 工程实践：给每个后台任务一个可恢复身份

对 OPC 来说，这类设计可以落到三个层面。

第一，任务身份要稳定。每个子 Agent run 需要 `run_id`、`parent_run_id`、`actor/tenant scope`、输入摘要和能力范围。主 Agent 不应靠“上一条消息里提到的那个任务”来恢复，而应该能用 ID 找回任务。

第二，进度要能被投影。给用户看的可以是“已完成资料收集，正在写正文”；给调度器看的可能是 `phase=write`, `milestone=research_collected`, `artifact_refs=[...]`；给审计看的则是用了哪些工具、访问了哪些资源、谁批准了什么。不同投影可以不同，但底层事件要统一。

第三，恢复要绑定 milestone。长任务失败时，系统不应该只会从头再跑。比如研究已完成、草稿已生成、只差发送 Telegram，那么恢复点应落在“发送前”而不是重新检索资料。否则重试会放大成本，还可能生成不一致结果。

## 取舍：太细会吵，太粗会瞎

progress signalling 的难点不是“能不能发事件”，而是粒度。如果每个 token、每个文件、每个 HTTP 请求都变成进度，用户会被噪音淹没，存储也会膨胀；如果只在开始和结束发事件，长任务仍然不可观察。

比较合理的边界是业务阶段和可恢复 artifact：凡是失败后能作为恢复点、审批点或用户判断点的地方，都值得成为 milestone；纯内部循环可以留在日志里。

另一个误区是把 progress 当成安慰剂 UI。真正有价值的进度不是“我还在努力”，而是“我已经完成什么、接下来做什么、如果中断可以从哪里继续”。这会直接影响用户信任和系统成本。

## 对 Dewey / OPC 的启发

如果 OPC 以后要做多 Agent 工作室，子 Agent 不应只是“派出去干活的人”，而应该是可观察、可暂停、可恢复、可审计的后台执行体。PM Agent、研究 Agent、代码 Agent、设计 Agent 都可以用同一套 progress/milestone 协议对外投影，只是展示层不同。

这也能让“柯南”作为 CTO-like coordinator 更稳：不是靠记忆里记住谁在做什么，而是读取一张任务运行图，看到每个子 Agent 的阶段、阻塞和产物。

一个值得先做的设计题是：OPC 的子 Agent milestone 最小集合应该是什么？如果只能保留 5 类事件，哪些事件足够支撑恢复、审计和用户可见进度？
